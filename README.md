## Описание

Скрипт проверяет актуальность версий всех компонентов Pangolin-стека, сравнивает их с установленными и при наличии обновлений:

1. Создаёт резервные копии конфигурационных файлов
2. Останавливает Docker-стек
3. Обновляет версии в `docker-compose.yml` и `traefik_config.yml`
4. Скачивает новые образы
5. Перезапускает стек
6. Удаляет устаревшие образы

Если все компоненты уже актуальны — завершается без изменений.

## Требования

- **OS:** Linux (Bash 4.0+)
- **Права:** root (или пользователь с доступом к Docker)
- **Зависимости:**

| Утилита | Назначение |
|---------|-----------|
| `curl` | Запросы к GitHub API и Docker Hub |
| `jq` | Парсинг JSON-ответов |
| `docker` | Управление контейнерами (`compose pull`, `up`, `down`) |
| `sed` | Замена версий в конфигурационных файлах |
| `awk` | Извлечение версии Badger из YAML |

Установка зависимостей на Debian/Ubuntu:

```bash
apt install curl jq docker.io
```

## Конфигурация

Все настройки находятся в верхней части скрипта в блоке **«Конфигурация»**:

```bash
PANGOLIN_DIR="/root/pangolin"                              # Рабочая директория стека
COMPOSE_FILE="${PANGOLIN_DIR}/docker-compose.yml"          # Путь к docker-compose.yml
TRAEFIK_CONFIG="${PANGOLIN_DIR}/config/traefik/traefik_config.yml"  # Конфиг Traefik
LOG_FILE="/var/log/pangolin-update.log"                    # Файл лога
LOCK_FILE="/tmp/pangolin-update.lock"                      # Lock-файл
LOG_MAX_LINES=1000                                         # Порог ротации лога
```

Измените переменные под своё окружение, если структура директорий отличается.

## Логика работы

```
Старт
  │
  ├─► Ротация лога (если > 1000 строк)
  ├─► Проверка зависимостей (curl, jq, docker, sed, awk)
  ├─► Захват lock-файла (защита от параллельного запуска)
  ├─► Чтение текущих версий из docker-compose.yml и traefik_config.yml
  ├─► Запрос актуальных версий через GitHub API и Docker Hub API
  ├─► Вывод таблицы сравнения версий
  │
  ├─[Все версии совпадают]──► Выход (код 0)
  │
  └─[Есть обновления]
        │
        ├─► Создание резервных копий конфигов (.bak.DDMMYYYY_HHMMSS)
        ├─► docker compose down
        ├─► sed: замена версий в docker-compose.yml и traefik_config.yml
        ├─► docker compose pull
        ├─► docker compose up -d
        └─► docker image prune -a -f
              │
              └─► Выход (код 0)
```

## Логирование

Лог записывается в `/var/log/pangolin-update.log`.

### Формат записи

```
[YYYY-MM-DD HH:MM:SS] [LEVEL] Сообщение
```

### Уровни

| Уровень | Цвет в терминале | Описание |
|---------|-----------------|----------|
| `INFO` | Голубой | Информационные сообщения о ходе работы |
| `OK` | Зелёный | Успешное завершение шага |
| `WARN` | Жёлтый | Предупреждения (например, устаревший lock-файл) |
| `ERROR` | Красный (stderr) | Критические ошибки, завершение скрипта |

> Цвета выводятся только в интерактивном терминале (TTY). При запуске через cron лог пишется без escape-последовательностей.

### Ротация лога

При превышении `LOG_MAX_LINES` (по умолчанию 1000 строк) текущий лог переименовывается:

```
/var/log/pangolin-update.log  →  /var/log/pangolin-update.log.YYYYMMDD
```

После чего создаётся новый пустой лог. Старые файлы не удаляются автоматически — при необходимости настройте `logrotate`.

### Просмотр лога

```bash
# Последние 50 строк
tail -50 /var/log/pangolin-update.log

# Только ошибки
grep '\[ERROR\]' /var/log/pangolin-update.log

# Все обновления
grep '\[OK\].*завершено' /var/log/pangolin-update.log
```

## Резервные копии

Перед каждым обновлением создаются копии конфигурационных файлов с суффиксом даты и времени:

```
docker-compose.yml              →  docker-compose.yml.bak.09042026_143205
config/traefik/traefik_config.yml  →  config/traefik/traefik_config.yml.bak.09042026_143205
```

### Восстановление из резервной копии

```bash
cd /root/pangolin

# Посмотреть доступные копии
ls -lt docker-compose.yml.bak.*

# Восстановить нужную версию
cp docker-compose.yml.bak.09042026_143205 docker-compose.yml

# Перезапустить стек
docker compose up -d
```

## Автоматизация через cron

Для ежедневной проверки обновлений в 3:00 ночи:

```bash
crontab -e
```

```cron
0 3 * * * /usr/local/bin/pangolin-update.sh
```

Для еженедельной проверки (в воскресенье в 4:00):

```cron
0 4 * * 0 /usr/local/bin/pangolin-update.sh
```

> При запуске через cron цвета в логе отключаются автоматически (нет TTY). Вывод в stdout/stderr можно перенаправить, но все ключевые сообщения и так пишутся в `LOG_FILE`.

---

## Коды выхода

| Код | Ситуация |
|-----|----------|
| `0` | Успех: обновление выполнено или не требуется |
| `1` | Ошибка: отсутствуют зависимости, недоступен API, не найден файл конфигурации, скрипт уже запущен |

## Структура файлов

```
/root/pangolin/
├── docker-compose.yml                    # Конфиг Docker Compose (обновляется скриптом)
├── docker-compose.yml.bak.DDMMYYYY_HHMMSS  # Резервная копия
└── config/
    └── traefik/
        ├── traefik_config.yml            # Конфиг Traefik/Badger (обновляется скриптом)
        └── traefik_config.yml.bak.DDMMYYYY_HHMMSS  # Резервная копия

/var/log/
├── pangolin-update.log                   # Текущий лог
└── pangolin-update.log.YYYYMMDD          # Архивный лог (после ротации)

/tmp/
└── pangolin-update.lock                  # Lock-файл (удаляется при завершении)

/usr/local/bin/
└── pangolin-update.sh                    # Сам скрипт
```
